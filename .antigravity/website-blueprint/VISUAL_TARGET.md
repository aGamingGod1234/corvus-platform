# Corvus Visual Target

## Desired effect

Corvus should feel calm enough for everyday work and exact enough for governed engineering. The first glance communicates three truths: whose workspace this is, where the agent will run, and how to start a thread. During execution, the flightpath makes authority and progress legible without turning the product into a monitoring dashboard.

## Visual hierarchy

1. Conversation and persistent composer.
2. Workspace identity, profile, and runtime truth.
3. Live flightpath: Plan, Work, Approval, Result.
4. Contextual artifacts, approvals, and evidence.
5. Navigation, search, schedules, and settings.

## Responsive targets

- Desktop `1440x1000`: 240px navigation, flexible conversation, inspector up to 360px; no horizontal overflow.
- Tablet `1024x900`: inspector collapses into a focus-restoring drawer; composer and run controls remain visible.
- Mobile `390x844`: thread-first reading order, compact identity, persistent composer, bottom navigation, full-screen sheets; 44px targets.

## Static success criteria

- The app is unmistakably Corvus through cool paper, graph rules, restrained editorial type, evidence mono, and the live flightpath.
- New thread and Send are visually primary without hiding provider, runtime, autonomy, or approval truth.
- Everyday and Developer views share identity but differ meaningfully in density and evidence.
- Team scope is labeled and capability-gated; color is supplementary.
- Empty, loading, offline, conflict, approval, and terminal states preserve direction.

## Motion success criteria

- Run progress is the one orchestrated motion moment.
- Drawers, menus, and inspector transitions maintain focus and do not compete with the run spine.
- Reduced motion removes travel while preserving state changes and announcements.
- Runtime evidence records active animations and reduced-motion matching.

## Failure conditions

- Generic SaaS KPI cards, a landing-page hero inside the product, copied T3Code/ChatGPT chrome, or ornamental AI gradients.
- Top-left profile switching or any client control that appears to grant authority.
- Cloud, collaboration, provider, billing, or offline claims that exceed advertised capabilities.
- Uncited visible sections or interactions, missing component hooks, stale approval hash, horizontal overflow, keyboard traps, or hidden error recovery.
