# Visual Target

## Desired Emotional Effect
Premium, specific, and source-backed. The first screenshot should feel intentional enough to belong in a serious portfolio, not like a generic generated template.

## First Viewport Requirements
- Category, offer, primary action, and tangible product/business proof must be visible.
- The hero must avoid generic SaaS dashboards and checklist-card layouts.
- If `PageOpening` exists, Hero starts after `site:opening-complete` or equivalent completed state.

## Visual Hierarchy
1. Specific headline and conversion action.
2. Concrete visual proof object.
3. Trust strip / objection reducers.
4. Secondary exploration action.

## Motion Style
Motion must feel premium and purposeful. `motionTaste` requires runtime evidence: recording, trace, animation timeline log, or reduced-motion verification.

## Static Screenshot Success Criteria
- Desktop 1440x1000: first viewport looks distinctive and conversion-ready.
- Tablet 1024x900: hierarchy remains intact with no awkward compression.
- Mobile 390x844: primary action and proof object remain visible without horizontal overflow.

## Failure Conditions
- Looks like a normal website mockup, generic SaaS dashboard, or checklist card section.
- Source influence is only listed but not visibly explainable.
- Motion is gimmicky or unverified.
