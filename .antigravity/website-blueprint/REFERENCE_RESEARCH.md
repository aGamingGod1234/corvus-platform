# Corvus Reference Research

- Research refreshed: 2026-07-16
- Required command sources: Corvus repository, T3Code, ChatGPT, shadcn Button, and Lucide Send.
- Direct HTTP blocks for the private Corvus repository and Cloudflare-protected ChatGPT page were resolved with authenticated GitHub and browser-readable evidence, recorded locally and hashed in `SOURCE_MANIFEST.json`.

## Findings

### Corvus repository

The authoritative product source establishes governed work, adaptive profiles, local-first continuity, truthful Cloud Preview status, and shared web/desktop contracts. It contributes product vocabulary and authority boundaries, not a layout to copy.

### T3Code

T3Code demonstrates a minimal multi-provider coding-agent surface and a desktop/web delivery shape. Corvus adopts the discipline of keeping agent work near the conversation and surfacing provider state contextually. Corvus does not copy T3Code chrome, spacing, theme, navigation, or implementation-specific branding.

### ChatGPT

ChatGPT demonstrates a stable new-chat entry, persistent history, a composer-led primary task, and progressive disclosure of secondary controls. Corvus adapts those conversion principles while adding explicit authority, runtime, autonomy, scheduling, and approval state. No wording, geometry, icons, or trade dress is copied.

### shadcn Button and Lucide Send

The shadcn Button source contributes semantic variants, focus-visible state, disabled state, and icon sizing conventions without adding its dependency. The Lucide Send source contributes the recognizable send glyph path, always paired with an accessible name. Both require visible `data-component-source` hooks.
