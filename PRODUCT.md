# Product

## Register

product

## Users

Developers and practical knowledge workers using a local desktop workspace to hand work to AI agents, inspect what ran, and keep execution governed. During the hackathon, the primary user is a developer demonstrating a complete local Codex workflow on Windows.

## Product Purpose

Corvus turns outcomes into governed, auditable local agent work. Success means a user can open the app, select a verified local model and reasoning level, start a safe run, understand its status, and manage the repositories, runs, schedules, and skills that support it without leaving the app.

## Brand Personality

Calm, capable, trustworthy. The interface should feel like a focused native tool: quiet when idle, precise around risk, and direct about unavailable capabilities.

## Anti-references

Avoid generic AI dashboards, decorative sci-fi treatments, nested card grids, inflated empty states, hidden security failures, and controls that appear interactive while their backing capability is unavailable.

## Design Principles

1. Keep the active task central and remove shell behavior that distracts from it.
2. Make safety and provider state legible, actionable, and honest.
3. Prefer familiar desktop-tool navigation and controls over novelty.
4. Reveal enough operational detail to build trust without burying the primary action.
5. Ship complete narrow workflows before broad preview surfaces.

## Accessibility & Inclusion

Target WCAG 2.1 AA for keyboard access, contrast, focus visibility, semantic status messages, and reduced motion. Never rely on color alone for run or safety state.
